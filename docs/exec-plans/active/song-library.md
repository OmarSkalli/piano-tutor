## Concrete Steps

### Milestone 1: Song Library route and component

1. Add `LibraryEntry` type to `src/types.ts`:

   ```typescript
   export interface LibraryEntry {
     id: string
     title: string
   }
   ```

2. Rewrite `src/routes/index.tsx`:

   ```typescript
   import { createFileRoute, Link } from '@tanstack/react-router'
   import library from '../songs/library.json'
   import type { LibraryEntry } from '../types'

   export const Route = createFileRoute('/')({
     component: SongLibrary,
   })

   function SongLibrary() {
     const songs = [...(library as LibraryEntry[])].sort((a, b) =>
       a.title.localeCompare(b, undefined, { sensitivity: 'base' }),
     )

     return (
       <main className="mx-auto max-w-lg p-6">
         <h1 className="mb-6 text-2xl font-semibold">Songs</h1>
         <ul className="divide-y">
           {songs.map((song) => (
             <li key={song.id}>
               <Link
                 to="/songs/$songId"
                 params={{ songId: song.id }}
                 className="block py-3 hover:text-blue-600"
               >
                 {song.title}
               </Link>
             </li>
           ))}
         </ul>
       </main>
     )
   }
   ```

3. Verify no type errors:

   ```bash
   pnpm exec tsc --noEmit
   ```

   Expected: no errors

4. Run dev server and confirm list appears:
   ```bash
   pnpm dev
   ```

### Milestone 2: Song route stub

## Idempotence and Recovery

### Safe to Re-run

- Editing route files is idempotent — the Vite plugin regenerates `routeTree.gen.ts` on each save
- No database or external state involved

### Rollback Procedure

```bash
git checkout -- src/routes/index.tsx src/types.ts
git rm -f src/routes/songs/$songId.tsx
```

### Known Risks

- **`library.json` import**: Vite handles JSON imports natively; no extra config needed
- **`routeTree.gen.ts` conflicts**: If the file has local edits, the plugin may fail to regenerate — ensure the file is unmodified before starting
