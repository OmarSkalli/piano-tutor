# Vision

Strategic direction and core principles for Piano Tutor.

## Mission

Make learning and practicing piano interactive and intuitive — from understanding a new song to playing it hands-on with real-time feedback.

## Goals

### Short-term (3-6 months)

- Import songs via MIDI and visualize them on a piano keyboard
- Build Study Mode: navigate a song note-by-note, understand the sheet, hear playback
- Build Practice Mode: connect a MIDI keyboard and get real-time feedback while playing

### Long-term (1-2 years)

- Support a growing library of songs with smooth import UX
- Expand Practice Mode with tempo control, looping sections, and progress tracking
- Become a go-to personal tool for learning new piano pieces

## Core Principles

**Interactive over passive**
Every feature should invite the user to engage — visualize, play, and get feedback — not just read or watch.

**MIDI-first**
MIDI is the source of truth for both song import and keyboard input. Keep the data model clean around it.

**Simple and focused**
This is a personal learning tool. Avoid feature bloat — each addition should directly support learning or practicing a song.

## Non-Goals

- Not a social platform or shared song library
- Not a general music theory course — focused on learning specific songs

## Success Metrics

- Can import a MIDI file and immediately start studying it
- Can play through a song in Practice Mode and receive accurate note feedback
- App feels fast and responsive on a desktop browser
